# REQUIREMENTS.md

# Functional Requirements

## Authentication

* User Signup
* User Login
* User Logout
* Role-Based Authorization

## Customer Features

* View Profile
* View Accounts
* View Balance
* Deposit Funds
* Withdraw Funds
* Transfer Funds
* View Transactions
* Apply for Loan

## Admin Features

* Add Customer
* Update Customer
* Delete Customer
* View Accounts
* View Transactions
* Generate Reports

## Manager Features

* View Loan Applications
* Approve Loans
* Reject Loans

---

# Non-Functional Requirements

## Security

* JWT Authentication
* Protected APIs
* Role-Based Access

## Performance

* Response time < 2 seconds

## Scalability

* Support future modules

## Reliability

* ACID-compliant transactions

## Maintainability

* Modular codebase
* Service-based architecture
